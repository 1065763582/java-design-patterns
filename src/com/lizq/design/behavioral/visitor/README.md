# 访问者模式(Visitor Pattern)

行为型

封装一些作用于某种数据结构中的各元素的操作，它可以在不改变数据结构的前提下定义作用于这些元素的新的操作。

#### 适用场景

业务规则要求遍历多个不同的对象，可以看作是迭代器模式的升级

#### 优点

增加新的行为容易，只需要增加一个访问者

#### 缺点

1. 具体元素变更困难，需要修改所有的访问者类
2. 访问者要访问一个类就必然要求这个类公布一些方法和数据，也就是说访问者关注了其他类的内部细节，违反了迪米特原则
